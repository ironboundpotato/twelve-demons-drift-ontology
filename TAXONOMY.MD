# The Twelve Demons of Drift
## A Formal Taxonomy of AI Reasoning Failure Modes
**Author: Kevin Gilbert**
**Version: 1.0**

> "Systems collapse when constraints fail." — Kevin Gilbert

---

## Overview

The Twelve Demons of Drift is a structured taxonomy of failure modes that cause AI systems to lose alignment, coherence, and operational integrity during extended interactions.

Each Demon represents a distinct class of destabilization. Each maps to a parallel failure mode in human systems — documented in the Driftwalker framework.

Drift is not random. It is structural. It follows predictable patterns.
Once named, it can be detected. Once detected, it can be governed.

---

## Severity Model

All Demons are classified using a four-level severity scale:

| Level | Name | Description |
|-------|------|-------------|
| L1 | Cosmetic | Minor deviation. No operational impact. Log only. |
| L2 | Functional | Output correctness affected. Clarification required. |
| L3 | Structural | Task reliability degraded. Containment required. |
| L4 | Critical | Mission-threatening destabilization. Escalation required. |

**Multi-Demon Rule:** When multiple Demons are active simultaneously, the highest severity governs the response.

## The Twelve Demons

---

### DEMON 1 — ROLE DRIFT
**Class:** Identity Failure
**Human Parallel:** Role Collapse — Loss of identity and direction

The system abandons its assigned role and begins improvising new ones without operator authorization.

The model stops being what it was asked to be.

| Severity | Trigger |
|----------|---------|
| L1 | Slight tone shift outside assigned role |
| L2 | Model adopts adjacent role without permission |
| L3 | Model operates in a completely different role |
| L4 | Model redefines its own identity and purpose |

---

### DEMON 2 — TONE DRIFT
**Class:** Expression Failure
**Human Parallel:** Emotional Collapse — Fragmentation and dissociation

The emotional and expressive register of the system shifts without operator authorization.

Formality collapses. Personality bleeds through constraints. The signal changes without the content changing.

| Severity | Trigger |
|----------|---------|
| L1 | Minor register variation within acceptable range |
| L2 | Noticeable tone shift affecting output quality |
| L3 | Persistent tone misalignment undermining trust |
| L4 | Complete emotional instability in outputs |

---

### DEMON 3 — FORMAT DRIFT
**Class:** Structure Failure
**Human Parallel:** Dissociation — Numbness and detachment from structure

Output deviates from required structure, headings, steps, length, or formatting constraints.

The container breaks. The content spills.

| Severity | Trigger |
|----------|---------|
| L1 | Minor formatting inconsistency |
| L2 | Required structure partially missing |
| L3 | Output structure unrecognizable from specification |
| L4 | Complete structural collapse across outputs |

---

### DEMON 4 — INTENT DRIFT
**Class:** Purpose Failure
**Human Parallel:** Meaning Collapse — Despair and misdirection

The system answers a different question than the operator asked.

Intent is what is being solved. When intent drifts, the system pursues its own interpretation of the problem rather than the operator's stated objective.

| Severity | Trigger |
|----------|---------|
| L1 | Minor reframing of question within acceptable range |
| L2 | Output addresses adjacent problem |
| L3 | Output addresses wrong problem entirely |
| L4 | System pursues self-generated agenda |

---

### DEMON 5 — SCOPE DRIFT
**Class:** Boundary Failure
**Human Parallel:** Burnout — Boundary violation and external control

Output expands, contracts, or shifts beyond defined operational boundaries.

Scope is how far the solution is allowed to extend. When scope drifts, the system exceeds or abandons its mandate.

| Severity | Trigger |
|----------|---------|
| L1 | Slight boundary expansion within tolerance |
| L2 | Noticeable scope expansion beyond specification |
| L3 | Significant boundary violation affecting task integrity |
| L4 | Complete scope collapse — system operates without boundaries |

---

### DEMON 6 — CONFIDENCE DRIFT
**Class:** Certainty Failure
**Human Parallel:** Self-Control Loss — Impulse and addiction to certainty

System certainty rises or collapses beyond what evidence supports.

Overconfidence produces hallucination. Underconfidence produces paralysis. Both are drift.

| Severity | Trigger |
|----------|---------|
| L1 | Minor calibration variance |
| L2 | Confidence inconsistent with output quality |
| L3 | Persistent overconfidence producing false assertions |
| L4 | Certainty completely decoupled from evidence |

---

### DEMON 7 — MEMORY DRIFT
**Class:** Continuity Failure
**Human Parallel:** Time Distortion — Regret and regression

Loss, corruption, or distortion of required context across a session or workflow.

The system forgets what it was told. Worse — it misremembers.

| Severity | Trigger |
|----------|---------|
| L1 | Minor context gap within acceptable range |
| L2 | Required context partially lost |
| L3 | Critical context missing affecting task integrity |
| L4 | Complete context corruption — system operates on false history |

---

### DEMON 8 — CONTEXT DRIFT
**Class:** Awareness Failure
**Human Parallel:** Dissociation — Loss of present awareness

The system loses accurate awareness of its current operational environment, constraints, or state.

Unlike Memory Drift (past), Context Drift affects present situational awareness.

| Severity | Trigger |
|----------|---------|
| L1 | Minor situational misread |
| L2 | Incorrect operational assumptions |
| L3 | System operates on fundamentally wrong context model |
| L4 | Complete loss of operational awareness |

---

### DEMON 9 — REASONING DRIFT
**Class:** Logic Failure
**Human Parallel:** Internal Conflict — Chaos and dissonance

Errors emerge in multi-step reasoning chains. Logic degrades. Conclusions detach from premises.

The system thinks it is reasoning correctly while its reasoning has broken down.

| Severity | Trigger |
|----------|---------|
| L1 | Minor logical inconsistency |
| L2 | Reasoning gap affecting output accuracy |
| L3 | Systematic reasoning failure across outputs |
| L4 | Complete logical collapse — conclusions contradict premises |

---

### DEMON 10 — GOAL DRIFT
**Class:** Direction Failure
**Human Parallel:** Agency Loss — Inaction and paralysis

The system's operational objectives shift away from the operator's stated goals.

Related to Intent Drift but distinct — Intent Drift is about interpretation of the problem. Goal Drift is about what the system is actually trying to achieve.

| Severity | Trigger |
|----------|---------|
| L1 | Minor goal reframing within tolerance |
| L2 | System pursues adjacent goal |
| L3 | System actively works toward wrong objective |
| L4 | System pursues self-generated goals against operator intent |

---

### DEMON 11 — ALIGNMENT DRIFT
**Class:** Values Failure
**Human Parallel:** Spiraling Anxiety — Intrusion and hypervigilance

The system's values, priorities, and ethical weight assignments shift from their established baseline.

Alignment Drift is the slow erosion of what the system cares about.

| Severity | Trigger |
|----------|---------|
| L1 | Minor value weighting variance |
| L2 | Priorities inconsistent with established baseline |
| L3 | Systematic value misalignment affecting decisions |
| L4 | Complete value inversion — system works against its alignment |

---

### DEMON 12 — HIJACK DRIFT
**Class:** Adversarial Failure
**Human Parallel:** Overload — Exhaustion and disconnection from self

The system is manipulated by external input into operating outside its constraints.

Prompt injection. Adversarial takeover. The system is no longer serving its operator — it is serving whoever hijacked it.

| Severity | Trigger |
|----------|---------|
| L1 | Minor susceptibility to leading questions |
| L2 | Output influenced by adversarial framing |
| L3 | System operating partially under external control |
| L4 — CRITICAL | Complete adversarial takeover — system serves hostile intent |

---

## Governance Response by Severity

| Severity | Pathway | Action |
|----------|---------|--------|
| L1 | Observability | Log and monitor |
| L2 | Clarification | Pause and verify with operator |
| L3 | Containment | Halt, stabilize, reduce autonomy |
| L4 | Escalation | Override, revoke, alert operator |

---

## Multi-Demon Co-occurrence

Demons frequently co-occur. Common dangerous combinations:

- **Role Drift + Goal Drift** — System has become something else pursuing something else
- **Confidence Drift + Reasoning Drift** — Certain about wrong conclusions
- **Hijack Drift + Alignment Drift** — External control eroding values
- **Memory Drift + Context Drift** — Complete loss of situational grounding
- **Intent Drift + Scope Drift** — Wrong problem, wrong boundaries

When multiple Demons are active, highest severity governs response.
